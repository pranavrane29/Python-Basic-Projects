##### **## Project Overview**

The Personal Expense Tracker is a Python console application that helps users track their daily 

expenses by category (Food, Travel, Entertainment, etc.), calculate monthly totals, compare 

expenses against a set budget, and identify overspending. The application uses lists of 

dictionaries to maintain a running expense log and applies string formatting for clear reporting. 

\-----------------------------------------------------------------------

##### 

##### **## Data Structure Used**

\*\*List - Used list to store the expenses made in ordered format so that it becomes easy to track

and make changes to data if any. Also used date list to store starting and ending dates.

\*\*Dictionary - Nested dictionary used in the list to store all the expense related values one by one

together. Also, used summary dictionary to store category wise spending.

\----------------------------------------------------------------------



##### **## Functions Used**



add\_expense() — Register a new expense made

view\_expenses()     — View all the expenses made withb details

category\_summary()    — Shows category wise summary of expenses made

budget\_report()    — Calculates budget report and gives warning if expenses went overbudget or so. 

top\_category()   — Showcases most expensive category

average\_daily\_spending() — Calculates average daily spending in specific dates

show\_menu() - Shows operation you can perform



\---------------------------------------------------------------------



##### **## Key Features**

\- Negative expense prevention on add

\- Modification on expenses made possible due to list DS.

\- Automatic Warning Sign when budget overhead

\- Tracks most expensive purchase
- Input validation via try-except on all numeric inputs

\- Empty record guard on most methods.

\- Clean output for all records



\------------------------------------------------------------------------



##### **## Bonus Feature — Calculate Daily Average spending**

Takes a date range from the user and calculates the daily 

average spending in that specific date period.
